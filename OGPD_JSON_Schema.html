Titel title X String Der Titel beschreibt den Datensatz, das Dokument oder die App prägnant und wird z. B. in Suchergebnissen und Listen angezeigt.
Bezeichner name X String [a-z0-9_-]{2;} Ein für Menschen lesbarer Bezeichner des Datensatzes, des Dokuments oder der Apps, der für eine Identifizierung genutzt werden kann (einfache ASCII-Präsentation des Titels, auch für den pretty-link)
Beschreibung notes X String Beschreibung und weitere Informationen zum Datensatz, zum Dokument oder zur App.
Kategorie group X Liste von String Enum aus: siehe Kategorien Tabelle Hier werden die Kategorien für die Datensätze und Dokumente aufgelistet. Die Kategorien sind statisch und werden vom Portalbetreiber langfristig gepflegt. Zusätzlich können mit subgroups fachspezifische Unterkategorien beispielsweise aus INSPIRE oder EVAS genutzt werden.
Schlagwörter tags Liste von String Mit frei wählbaren Schlagwörtern können hier Datensätze, Dokumente und Apps beschrieben werden, um diese leichter mit der Suche ausfindig machen zu können.
Webadresse zu weiteren Informationen über den Datensatz url String URI Die ursprüngliche Webadresse des Datensatzes, Dokuments oder der App, um zu weiteren Informationen über den Datensatz, das Dokument oder die App zu gelangen.
Ressourcen resources X Liste von Object Folgende Felder können für jede Ressource individuell angegeben werden:
URL url X String URI Verweis auf die eigentliche Datendatei, das Dokument oder die Anwendung.
Format format X String Typ der Ressource (MIME Types)
Beschreibung description String Erläuterung, welche Rolle die Ressource für den Datensatz, das Dokument bzw. die App spielt (z. B. ob es sich dabei um eine bestimmte Zeitscheibe oder Schlüsselliste handelt.)
Sprache language String [a-z]{2} Sprache als ISO 639-1-Code, in der die Ressource verfasst ist.
Überprüfung hash String Bei statischen Dateien die SHA2-Prüfsumme bzw. bei dynamischen Daten ein Zertifikat zur Überprüfung der Integrität
Extras Folgende Felder können für jeden Datensatz individuell angegeben werden:
Kontaktliste contacts X Liste von Object Liste der Kontakte zu Metadaten, Daten, Dokument bzw. App, mindestens ein Kontakt ist anzugeben, bevorzugt der Ansprechpartner
Name name String Person oder Funktion
Adressinformation address  String
Website url String URI
Email email String
Rolle role String Enum aus: veroeffentlichende_stelle, ansprechpartner, vertrieb, autor Rolle, die dieser Kontakt für den Datensatz spielt, z.B. Autor, Vertrieb, Veröffentlichende Stelle, Ansprechpartner
Daten dates X Liste von Object Erstellungs-, Veröffentlichungs- und Aktualisierungsdaten von Daten, Dokumenten, Apps und ihren Metadaten
Rolle role String Enum: erstellt, veroeffentlicht, aktualisiert Beschreibt um was für ein Datum es sich handelt
Datum date String date-time das Datum
Nutzungsbestimmungen terms_of_use X Festlegung der spezifischen Nutzungsbestimmungen des Datensatzes, des Dokuments oder der App.
Lizenz-Bezeichnung license_id String Enum aus: LizenzID Tabelle Lizenz aus einer festen Liste
URL Lizenz-Bezeichnung license_url String URI URL, auf der die Lizenz des Datensatzes, Dokuments oder der App beschrieben und erklärt ist
Freitext zur Beschreibung other String Freitext, der die Nutzungsbestimmungen festlegt, falls diese keiner allgemein bekannten Lizenz entsprechen.
Nutzungsfreiheit is_free_to_use String URI Die Information, ob der Datensatz/das Dokument/die App kostenfrei und zweckoffen ist (muss gesetzt werden, wenn Freitext bei der Lizenzangabe genutzt wird)
Typ des Inhalts content_type X String Enum: datensatz, dokument, app technisches Feld, welches definiert ob sich der Metadateneintrag auf einen Datensatz, ein Dokument oder eine App bezieht.
Unterkategorien subgroups Liste von String Hier werden die Unterkategorien für die Datensätze und Dokumente aufgelistet. Im Gegensatz zu den Kategorien werden diese genau den fachlichen Kategorien der Datenbereitsteller angeglichen, d.h. jeder neu angeschlossene Datenbereitsteller kann hier benötigte Unterkategorien hinzufügen.
Original-Metadaten-Portal metadata_original_portal String URI URL des Portals, von dem der Metadateneintrag des Datensatzes oder des Dokuments geharvestet wurde.
Original-Metadaten-Schlüssel metadata_original_id String Der Identifier des ursprünglichen Metadateneintrags der Datensätze oder der Dokumente. Damit kann schnell die Originalposition von geharvesteten Datensätzen oder Dokumenten nachvollzogen und Dubletten erkannt werden. ["technisches"Feld] !UUID!
URL des Original-Metadaten-Eintrags auf die XML-Strukturen metadata_original_xml String URI URL des Portals, von dem der Metadateneintrag des Datensatzes oder des Dokuments geharvestet wurde. ["technisches" Feld]!XML!
URL auf Repräsentation des Original-Metadaten-Eintrags metadata_original_html String URI Präsentations-URL des Portals, von dem der Metadateneintrag des Datensatzes oder des Dokuments geharvestet wurde. [„technisches“  Feld]!HTML!
Geographische Abdeckung in Koordinaten spatial Object Die geographische Abdeckung des Datensatzes in WGS 84 Koordinaten nach GeoJSON http://geojson.org
type String Enum: Polygon Art der Form
coordinates Array aus NumberArrays [x,y] Liste der Koordinaten. Koordinaten wiederum Arrays in der Reihenfolge x, y ( [[x1,y1], [x2,y2])
Geographische Abdeckung als Text spatial-text String Abgedecktes Gebiets, wenn möglich der Amtliche Gemeindeschlüssel
Geographische Granularität geographical_granularity String Enum: bund, land, kommune, Stadt Die geographische Granularität des Datensatzes, eines Dokuments, einer App
Zeitraum von temporal_coverage_from String date-time Der Zeitpunkt, von dem an der Datensatz, das Dokument oder die App einschließlich Daten enthält
Zeitraum bis temporal_coverage_to String date-time Der Zeitpunkt, bis zu dem der Datensatz, das Dokument oder die App einschließlich Daten enthält
zeitliche Granularität temporal_granularity String Enum: sekunde, minute, stunde, tag, woche, monat, quartal, jahr, 5-jahre Die zeitliche Auflösung der enthaltenen Daten, des Dokuments oder der App
URL zu verwendeten Daten used_datasets StringArray URI Die verwendeten Datensätze von Dokumenten und Apps mithilfe einer Liste mit URLs auf Metadaten
Sektor sector String Enum: oeffentlich, privat, andere Zeigt an, ob eine App oder ein Dokument aus dem öffentlichem, dem privaten oder einem anderen Bereich kommt. [Feld wird nur für Dokumente und Apps genutzt]
OGDD Version ogdd_version String date-time OGDD Version des Datensatzes
